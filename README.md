# Инструкция по запуску проекта

Этот проект использует Docker для создания переносимой и изолированной среды. Следуйте этим шагам, чтобы запустить проект на вашем компьютере.

## Предварительные требования

*   [Docker](https://docs.docker.com/get-docker/)
*   [Docker Compose](https://docs.docker.com/compose/install/)

## Установка и запуск

1.  **Клонируйте репозиторий:**

    ```bash
    git clone <URL вашего репозитория>
    cd <имя папки репозитория>
    ```

2.  **Настройте переменные окружения:**

    Перейдите в папку `clod` и скопируйте файл `.env.example` в `.env`:

    ```bash
    cd clod
    cp .env.example .env
    ```

    При необходимости вы можете отредактировать файл `.env` и изменить значения переменных (например, пароли или пути к данным).

3.  **Соберите и запустите контейнеры:**

    В папке `clod` выполните следующую команду:

    ```bash
    docker-compose up --build -d
    ```

    Эта команда соберет образ вашего Python-приложения и запустит все сервисы в фоновом режиме.

4.  **Проверьте, что все работает:**

    Вы можете посмотреть логи запущенных контейнеров:

    ```bash
    docker-compose logs -f
    ```

    Или проверить статус контейнеров:

    ```bash
    docker-compose ps
    ```

## Остановка проекта

Чтобы остановить все запущенные сервисы, выполните команду:

```bash
docker-compose down
```

## Описание сервисов

*   `app`: Ваше основное Python-приложение.
*   `postgres_app`: База данных PostgreSQL для вашего приложения.
*   `postgres_temporal`: База данных PostgreSQL для Temporal.
*   `temporal`: Рабочий процесс Temporal.
*   `temporal-ui`: Веб-интерфейс для Temporal.
*   `minio`: S3-совместимое хранилище объектов.
*   `minio-client`: Клиент для Minio.
*   `portainer`: Веб-интерфейс для управления Docker.
