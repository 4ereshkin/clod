# Проект обработки облаков точек

Этот проект представляет собой конвейер для обработки данных облаков точек (Point Cloud), полученных, например, с помощью лидара (LiDAR). Он автоматизирует процесс загрузки, преобразования, хранения и извлечения данных для дальнейшего анализа и визуализации.

## Основные возможности

- **Загрузка данных**: Выбор и загрузка файлов с облаками точек (например, в формате LAS).
- **Извлечение метаданных**: Автоматическое извлечение и сохранение метаданных из исходных файлов.
- **Перепроецирование**: Изменение системы координат облака точек.
- **Хранение данных**: Загрузка обработанных данных в базу данных PostgreSQL с расширением pgpointcloud.
- **Извлечение данных**: Выгрузка данных из базы данных обратно в файлы.
- **Создание 3D-тайлов**: Подготовка данных для визуализации в веб-приложениях, таких как Cesium.

## Технологии

- **Python**: Основной язык разработки.
- **PDAL (Point Data Abstraction Library)**: Ключевая библиотека для чтения, записи и обработки облаков точек.
- **PostgreSQL + PostGIS/pgpointcloud**: Система управления базами данных для эффективного хранения больших объемов данных облаков точек.
- **py3dtiles**: Библиотека для создания 3D-тайлов, совместимых с Cesium.
- **Tkinter**: Используется для создания простого графического интерфейса для выбора файлов.
- **Pandas**: Применяется для работы с табличными данными, например, с записями о загрузках.

## Структура проекта

-   `clod/` - Основная директория проекта.
    -   `main.py`: Главный файл, определяющий и запускающий конвейер обработки.
    -   `gateway.py`: Отвечает за выбор файлов и извлечение исходных метаданных.
    -   `reproject.py`: Выполняет перепроецирование облаков точек.
    -   `insert.py`: Загружает данные в базу данных.
    -   `storage.py`: Содержит базовый класс для работы с базой данных.
    -   `records.py`: Читает записи о загрузках из базы данных.
    -   `fetch.py`: Извлекает облака точек из базы данных.
    -   `cesium.py`: Конвертирует облака точек в 3D-тайлы для Cesium.
    -   `requirements.txt`: Список зависимостей Python.
    -   `db.json`: Файл конфигурации для подключения к базе данных (пример).
-   `point_cloud/`: Директория для хранения исходных данных облаков точек (предположительно).
-   `dsc/`: Назначение этой директории неясно из кода.

## Использование

### 1. Установка зависимостей

Перед запуском проекта необходимо установить все зависимости, перечисленные в файле `requirements.txt`:

```bash
pip install -r clod/requirements.txt
```

### 2. Настройка базы данных

Убедитесь, что у вас установлен и настроен PostgreSQL с расширениями PostGIS и pgpointcloud. Создайте файл `clod/db.json` по аналогии с `clod/db.example.json` и укажите в нем свои учетные данные для подключения к базе данных.

### 3. Запуск конвейера

Основная логика конвейера находится в файле `clod/main.py`. Вы можете настраивать последовательность шагов, создавая и изменяя экземпляры класса `Orchest`.

**Пример:**

```python
# clod/main.py

# ... (импорты и определения классов)

# Конвейер для загрузки и перепроецирования данных
orchest = Orchest([
    StepCheckpoint(checkpoint=Checkpoint()),
    StepReproject(in_srs='EPSG:4490', out_srs='EPSG:4326')
])

# Конвейер для создания 3D-тайлов
cesium = Orchest([
    StepCheckpoint(checkpoint=Checkpoint()),
    StepReproject(in_srs='EPSG:4326', out_srs='EPSG:4978'),
    StepCesiumTiles(cesium_cls=CesiumImport)
])

# Запуск одного из конвейеров
cesium.run()
```

Вы можете раскомментировать или изменить вызов `.run()` для запуска нужного вам конвейера.
